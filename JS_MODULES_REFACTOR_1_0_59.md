# JS Modules Refactor 1.0.59

تم تقسيم `js/index_script.js` إلى ملفات مصدر وظيفية داخل `js/modules/` مع إبقاء التشغيل متوافقًا عبر loader صغير في `js/index_script.js`.

## سبب اختيار Loader موحد
التطبيق يستخدم ES module لاستيراد Firebase، وفي الوقت نفسه توجد دوال كثيرة تعتمد على نفس الحالة العامة. لذلك تم تقسيم المصدر إلى ملفات وظيفية ثم تحميلها وتجميعها بنفس الترتيب داخل runtime واحد، حتى لا تتكسر المتغيرات المشتركة أو دوال الصفحات.

## الملفات الجديدة
- `js/modules/00_core_update_auth.js`: النواة العامة، الإصدارات، الحوارات، التحديث، المصادقة المساعدة والتنبيهات (772 سطر تقريبًا)
- `js/modules/10_firebase_live_data.js`: تهيئة Firebase والمستمعات اللحظية والبيانات العامة (243 سطر تقريبًا)
- `js/modules/20_router_setup_profile.js`: محرك العرض والتنقل وتهيئة الحساب والربط والصفحات العامة (876 سطر تقريبًا)
- `js/modules/30_purchase_catalog.js`: شراء العميل والكتالوج والسلة ونموذج الطلب (401 سطر تقريبًا)
- `js/modules/40_pages_tables.js`: صفحات الأصناف والعملاء والطلبات والفواتير والجداول الأساسية (213 سطر تقريبًا)
- `js/modules/50_auth_order_approval.js`: استقرار تسجيل الدخول والطلب واعتماد التاجر (501 سطر تقريبًا)
- `js/modules/60_payments_returns_policies.js`: السداد والفواتير والمرتجعات والأقساط والسياسات (501 سطر تقريبًا)
- `js/modules/70_settings_owner_items_bridge.js`: الإعدادات والمالك والأصناف والأسعار والمخزون وجسر أندرويد والتشغيل (601 سطر تقريبًا)

## ما تم فحصه
- فحص تركيب الملفات المجمعة.
- فحص تركيب `js/index_script.js` loader.
- فحص `android-update.json`.
- تحديث `APP_VERSION` و `APP_BUILD_CODE` داخل runtime.
- تحديث `versionCode/versionName`.
- تحديث روابط `index.html` إلى 1.0.59.
- الحفاظ على نفس منطق التطبيق بدون إضافة ميزات جديدة.
