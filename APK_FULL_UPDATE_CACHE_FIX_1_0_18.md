# حسابي التجاري 1.0.18

## الهدف
إصلاح مشكلة أن تحديث APK يثبت التطبيق لكن الواجهات تبقى قديمة أو تختفي التحديثات بعد إغلاق التطبيق، ولا تظهر إلا عند الضغط على زر تحديث الواجهات.

## ما تم
- دمج تعديلات 1.0.17 الخاصة بنظام الجداول حسب الطلب.
- رفع إصدار الواجهات إلى 1.0.18.
- رفع Android versionCode إلى 18 وversionName إلى 1.0.18.
- تحديث android-update.json إلى 1.0.18.
- جعل زر تحديث APK يمسح Cache API وService Worker قبل تنزيل APK.
- إضافة دالة Android Native لتنظيف WebView cache عند بدء التحديث.
- عند فتح التطبيق بعد تثبيت APK جديد، يتم اكتشاف تغير versionCode ثم تنظيف WebView cache وفتح الرابط مع رقم الإصدار الجديد لمنع تحميل واجهة قديمة.

## النتيجة المتوقعة
بعد تثبيت APK الجديد، يجب أن تفتح آخر واجهات التطبيق تلقائيًا بدون الضغط على زر تحديث الواجهات. زر تحديث الواجهات يبقى احتياطيًا فقط.
